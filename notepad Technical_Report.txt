EXECUTIVE SUMMARYThis project implements a comprehensive e-commerce analytics platform using three complementary big data technologies: MongoDB (document database), HBase (wide-column store), and Apache Spark (distributed processing). The system analyzes synthetic e-commerce data containing **1,000 users, 500 products, 5,000 sessions, and 538 transactions** spanning 90 days of activity.**Key Achievements:**- Designed optimized schemas for MongoDB and HBase based on specific query patterns- Implemented complex aggregation pipelines in MongoDB for revenue analytics- Created time-series data models in HBase with optimized row key design (user_id + reverse_timestamp)- Developed distributed data processing pipelines using Spark SQL- Generated 7 professional visualizations revealing actionable business insights**Key Business Insights:**- **Texas** generates highest revenue ($47,863) but **Pennsylvania** has highest average order value ($847)- **December** is peak sales month ($128,466), 74% higher than February- **30.48%** of browsing sessions convert to purchases (excellent for e-commerce)- Category 6 products contribute 15.2% of total revenue- Top customer (user_000652) spent $8,590 from just 4 orders---## 2. PART 1: DATA MODELING AND STORAGE### 3.1 MongoDB Implementation#### 3.1.1 Data Modeling Decisions**Users Collection:**```json{&nbsp; "user\_id": "user\_000001",&nbsp; "geo\_data": {&nbsp;   "city": "Austin",&nbsp;   "state": "TX",&nbsp;   "country": "US"&nbsp; },&nbsp; "registration\_date": "2025-12-15T08:42:13",&nbsp; "last\_active": "2026-02-22T16:33:57"}Design Rationale: Embedded geo\_data keeps location information together with user profile, avoiding separate table joins. Indexes created on user\_id (unique) and geo\_data.state for state-based aggregations.Products Collection:{&nbsp; "product\_id": "prod\_00279",&nbsp; "name": "Devolved 3rdgeneration time-frame",&nbsp; "category\_id": "cat\_010",&nbsp; "price": 186.23,&nbsp; "current\_stock": 342,&nbsp; "status": "active",&nbsp; "specifications": {&nbsp;   "brand": "TechCorp",&nbsp;   "weight": "5 kg"&nbsp; }}Design Rationale: Category referenced rather than embedded to maintain flexibility. Embedded specifications for product attributes. Indexes on product\_id, category\_id, and price.Transactions Collection:{&nbsp; "transaction\_id": "txn\_a1b2c3d4e5f6",&nbsp; "user\_id": "user\_000652",&nbsp; "timestamp": "2025-12-20T14:52:41",&nbsp; "items": \[&nbsp;   {&nbsp;     "product\_id": "prod\_00279",&nbsp;     "quantity": 2,&nbsp;     "price\_per\_unit": 186.23,&nbsp;     "subtotal": 372.46&nbsp;   }&nbsp; ],&nbsp; "total": 754.80,&nbsp; "payment\_method": "credit\_card",&nbsp; "status": "delivered"}Design Rationale: Embedded items array preserves line items within transaction. Denormalized price captures value at purchase time. Indexes on transaction\_id, user\_id, and timestamp.\#### 3.1.2 MongoDB Aggregation Pipelines\*\*Query 1: Top 10 Products by Revenue\*\*```javascriptdb.transactions.aggregate(\[&nbsp; {$unwind: "$items"},&nbsp; PART 2: DATA PROCESSING WITH APACHE SPARK4.1 Spark Session Initializationfrom pyspark.sql import SparkSessionspark = SparkSession.builder \\&nbsp;   .appName("ECommerce Analytics") \\&nbsp;   .config("spark.sql.adaptive.enabled", "true") \\&nbsp;   .getOrCreate()4.2 Data Loadingusers\_df = spark.read.json("users.json")products\_df = spark.read.json("products.json")transactions\_df = spark.read.json("transactions.json")sessions\_df = spark.read.json("sessions.json")users\_df.createOrReplaceTempView("users")products\_df.createOrReplaceTempView("products")transactions\_df.createOrReplaceTempView("transactions")sessions\_df.createOrReplaceTempView("sessions")4.3 Spark SQL AnalyticsQuery 1: Customer Lifetime ValueSELECT &nbsp;   u.user\_id,&nbsp;   u.geo\_data.state,&nbsp;   COUNT(DISTINCT t.transaction\_id) as num\_orders,&nbsp;   ROUND(SUM(t.total), 2) as total\_spent,&nbsp;   ROUND(AVG(t.total), 2) as avg\_order\_valueFROM users uJOIN transactions t ON u.user\_id = t.user\_idGROUP BY u.user\_id, u.geo\_data.stateORDER BY total\_spent DESCLIMIT 10Results:user\_id	state	orders	total\_spent	avg\_orderuser\_000652	GA	4	$8,590.00	$2,147.50user\_000994	PA	6	$8,248.64	$1,374.77user\_000052	FL	9	$7,081.69	$786.85user\_000820	TX	3	$6,391.77	$2,130.59user\_000969	NC	3	$5,965.48	$1,988.49user\_000060	CA	2	$5,868.92	$2,934.46user\_000914	TX	4	$5,758.66	$1,439.67user\_000854	GA	4	$5,527.92	$1,381.98user\_000078	NY	7	$4,942.94	$706.13user\_000509	NC	6	$4,705.35	$784.23Query 2: Product PerformanceWITH product\_sales AS (&nbsp;   SELECT &nbsp;       explode(items) as item&nbsp;   FROM transactions)SELECT &nbsp;   p.product\_id,&nbsp;   p.name,&nbsp;   p.category\_id,&nbsp;   COUNT(DISTINCT ps.transaction\_id) as times\_purchased,&nbsp;   SUM(ps.item.quantity) as total\_quantity,&nbsp;   ROUND(SUM(ps.item.subtotal), 2) as total\_revenueFROM product\_sales psJOIN products p ON ps.item.product\_id = p.product\_idGROUP BY p.product\_id, p.name, p.category\_idORDER BY total\_revenue DESCLIMIT 10Results:product\_id	name	category	purchases	quantity	revenueprod\_00279	Devolved 3rdgeneration time-frame	cat\_010	8	16	$6,199.97prod\_00166	Enhanced hybrid software	cat\_015	5	14	$6,014.72prod\_00093	Networked actuating array	cat\_009	5	15	$5,847.24Query 3: Session Conversion FunnelSELECT &nbsp;   conversion\_status,&nbsp;   COUNT(\*) as num\_sessions,&nbsp;   ROUND(COUNT(\*) \* 100.0 / (SELECT COUNT(\*) FROM sessions), 2) as percentageFROM sessionsGROUP BY conversion\_statusORDER BY num\_sessions DESCResults:conversion\_status	num\_sessions	percentageadded\_to\_cart	1,601	32.02%viewed	1,551	31.02%converted	1,524	30.48%bounce	324	6.48%Query 4: Monthly Revenue TrendSELECT &nbsp;   DATE\_FORMAT(TO\_DATE(timestamp), 'yyyy-MM') as month,&nbsp;   COUNT(\*) as transactions,&nbsp;   ROUND(SUM(total), 2) as revenueFROM transactionsGROUP BY DATE\_FORMAT(TO\_DATE(timestamp), 'yyyy-MM')ORDER BY monthResults:month	transactions	revenue2025-11	45	$30,863.912025-12	176	$128,466.522026-01	198	$127,151.082026-02	119	$73,214.10Query 5: State-wise PerformanceSELECT &nbsp;   u.geo\_data.state,&nbsp;   COUNT(DISTINCT u.user\_id) as total\_users,&nbsp;   COUNT(DISTINCT t.transaction\_id) as total\_transactions,&nbsp;   ROUND(COALESCE(SUM(t.total), 0), 2) as total\_revenue,&nbsp;   ROUND(COALESCE(AVG(t.total), 0), 2) as avg\_transaction\_valueFROM users uLEFT JOIN transactions t ON u.user\_id = t.user\_idGROUP BY u.geo\_data.stateORDER BY total\_revenue DESCResults:state	users	transactions	revenue	avg\_orderTX	105	65	$47,863.11	$736.36NC	119	68	$47,034.52	$691.68NY	95	74	$46,524.77	$628.71GA	86	62	$44,907.35	$724.31PA	91	48	$40,693.78	$847.79FL	99	66	$40,249.78	$609.85MI	111	47	$28,610.82	$608.74IL	100	50	$24,928.07	$498.56CA	97	29	$21,859.30	$753.77OH	97	29	$17,024.11	$587.045\.
